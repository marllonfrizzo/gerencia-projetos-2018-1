1. Preparação

1.1 Entender o Problema
Ir ao ambiente em que o sistema irá rodar e obter informações relevantes com o stackeholder.

1.2 Levantamento de Requisitos
Junto ao stakeholder, levantar informações sobre o funcionamento do negócio e assimilar ao desenvolvimento do sistema.

1.3 Modelagem de Dados
De posse das informações, modelar os dados e criar os diagramas. Para a documentação do sistema e para a arquitetura do sistema.

1.4 Escolha da Tecnologia
Analisar os requisitos e os diagramas e com base escolher qual é a tecnologia adequada para o projeto.

2 Execução/Desenvolvimento

2.1 Prototipagem
Esboçar o design e layout da aplicação.

2.2 Desenvolvimento
Escrever o código do programa.

2.3 Testes
Expor o sistema a diversas situações e observar seu comportamento.

2.4 Validação dos Requisitos
Com o sistema em funcionamento, validar com os requisitos previamente definidos.

3 Implantação

3.1 Implantação
Instalar o sistema no ambiente de produção.

3.2 Testes
Observar o comportamento do sistema, na intenção de encontrar falhas.

3.3 Validação com o Cliente
Verificar com o cliente se o sistema esta atendendo as suas necessidades.

3.4 Gerar Relatórios de Satisfação
Junto ao cliente, gerar relatórios quanto a sua satisfação com o sistema.

4 Manutenção

4.1 Manutenção
Depois do sistema estar em ambiente de produção, dar o devido suporte para eventuais problemas que possam ocorrer.
